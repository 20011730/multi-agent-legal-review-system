# LexRex AI - 디자인 시스템 가이드

## 디자인 키워드

### 핵심 컨셉
- **신뢰감**: 법률/기업 서비스의 권위 있는 이미지
- **정돈됨**: 명확한 정보 계층과 체계적인 레이아웃
- **분석적**: 데이터와 근거 중심의 구조화된 정보 전달
- **깔끔함**: 충분한 여백과 깨끗한 카드 디자인
- **전문성**: 너무 딱딱하지 않지만 프로페셔널한 톤앤매너

## 색상 팔레트

### 주요 색상
- **Primary (블루)**: `#1e40af` - 신뢰감과 전문성을 나타내는 메인 컬러
- **배경**: `#fafbfc` - 부드러운 오프화이트 배경
- **카드**: `#ffffff` - 순백 카드로 정보 강조
- **그레이디언트**: `from-slate-50 via-blue-50/30 to-slate-50` - 미묘한 블루 억양

### 에이전트별 색상 (멀티 에이전트 시각적 구분)
- **법률 전문가**: 블루 계열 (`blue-600`, `blue-50`)
- **리스크 관리자**: 에메랄드 계열 (`emerald-600`, `emerald-50`)
- **윤리 검토자**: 바이올렛 계열 (`violet-600`, `violet-50`)

### 상태 색상
- **승인/긍정**: 그린 계열 (`green-600`, `green-50`)
- **경고/조건부**: 앰버 계열 (`amber-600`, `amber-50`)
- **위험/재검토**: 레드 계열 (`red-600`, `red-50`)

## 타이포그래피

### 계층 구조
- **H1**: 서비스 타이틀 - 로고 영역
- **H2**: 페이지 메인 타이틀 (3xl, semibold)
- **H3**: 섹션 제목 (xl-2xl, semibold)
- **H4**: 카드 타이틀 (base-lg, medium)
- **Body**: 본문 텍스트 (sm-base, normal)
- **Caption**: 부가 설명 (xs, gray-500)

### 폰트 스타일
- **일반 텍스트**: 기본 시스템 폰트 (가독성 우선)
- **코드/원문**: `font-mono` - 검토 대상 원문 표시

## 레이아웃 구조

### 화면 구성
1. **고정 헤더**: 로고 + 주요 액션
2. **컨텐츠 영역**: 최대 너비 제한 (max-w-5xl ~ 7xl)
3. **카드 기반**: 정보를 카드 단위로 그룹화
4. **여백**: 충분한 padding과 gap (6, 8, 12 단위)

### 카드 유형
- **정보 카드**: 기본 border + 호버 shadow
- **상태 카드**: 색상 코딩된 border-l-4 accent
- **비교 카드**: 좌우 대비 (원본 vs 수정안)
- **타임라인 카드**: 순차적 메시지 흐름

## 컴포넌트 패턴

### 아이콘 사용
- **lucide-react** 사용 일관성
- 각 에이전트/상태별 전용 아이콘
- 크기: 4-6 (16-24px) 범위

### 배지
- **분석**: Secondary variant
- **쟁점**: Destructive variant
- **권고**: Green custom
- **리스크 레벨**: 색상 코딩 (high/medium/low)

### 버튼
- **Primary Action**: `bg-blue-600` + shadow
- **Secondary**: `variant="outline"`
- **크기**: 일반적으로 default, 중요 액션은 lg

## 정보 밀도

### 적절한 밀도 유지
- **카드 내부**: p-6~8
- **섹션 간격**: mb-6~8
- **텍스트 라인 높이**: leading-relaxed
- **리스트**: space-y-2~4

### 가독성 우선
- 한 화면에 너무 많은 정보 X
- 중요 정보는 시각적 강조
- 스크롤 가능한 영역은 명확히 구분

## 인터랙션

### 피드백
- **로딩**: Progress bar + Loader2 아이콘
- **완료**: CheckCircle + 색상 변경
- **에러/경고**: AlertTriangle + 앰버/레드

### 애니메이션
- **카드 진입**: `animate-in fade-in slide-in-from-bottom`
- **호버**: `hover:shadow-lg transition-all`
- **스케일**: 활성 요소 `scale-105`

## 사용자 경험 흐름

### 단계별 진행
1. **Home**: 서비스 소개 + 신뢰감 전달
2. **Input**: 명확한 폼 + 유의사항 안내
3. **Result**: 실시간 분석 진행 시각화
4. **Verdict**: 종합 보고서 + 실행 가능한 권고

### 일관된 네비게이션
- 헤더에 항상 로고 (클릭 시 홈)
- "뒤로 가기" 버튼 제공
- 진행 상황 명확히 표시

## 실무 적용 포인트

### 발표 시 고려사항
- 대형 화면에서도 가독성 유지
- 색상 대비 충분히 확보
- 핵심 정보 시각적 하이라이트

### 신뢰성 강화 요소
- 법률 조항 인용 (Badge로 표시)
- 단계별 프로세스 명시
- 에이전트 역할 명확히 구분
- 면책 조항 및 유의사항 강조
